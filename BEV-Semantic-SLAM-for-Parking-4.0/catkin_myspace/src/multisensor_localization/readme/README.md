## 如何运行  
启动start.sh脚本文件即可



## 工程规范
+ **加速编译**
1. 头文件包含尽可能精准
2. 系统库、cmake 指定库使用<>,自定义库使用"" 
3. 编译时的警告尽可能处理  

+ **代码规范**
1. 文件注释、函数注释注明  
2. 及时上传仓库  
3.  禁止省略for{}  

+ **调试规范**  
1. yaml读参务必录入LOG确认
2. 文件创建务必再次确认校验,并对异常处理
3. 调试运行时务必切换到Release模式下  
4. 利用脚本启动时,先启动节点再启动roscore，或可模拟实现并行启动

+ **开发规范**
1. 重大更新务必git传至仓库
2. 版本更新务必更新必要的说明
